# Spring框架总结

Spring框架是Java企业级开发的事实标准，提供了全面的企业级应用开发解决方案。本文档总结了Spring生态系统的核心概念和面试重点。

## 核心面试重点

### 1. IoC容器
- **控制反转**：将对象的创建和依赖关系的管理交给容器
- **依赖注入**：通过构造函数、setter方法或字段注入依赖
- **Bean生命周期**：实例化 → 属性注入 → 初始化 → 使用 → 销毁
- **作用域**：singleton、prototype、request、session、application

### 2. AOP面向切面编程
- **核心概念**：切面、连接点、切点、通知、目标对象、代理
- **代理模式**：JDK动态代理、CGLIB代理
- **通知类型**：前置通知、后置通知、环绕通知、异常通知、最终通知
- **切点表达式**：execution、within、this、target、args

### 3. Spring MVC
- **核心组件**：DispatcherServlet、HandlerMapping、HandlerAdapter、ViewResolver
- **请求流程**：请求 → DispatcherServlet → HandlerMapping → HandlerAdapter → Controller → ViewResolver → View
- **数据绑定**：@RequestParam、@PathVariable、@RequestBody、@ModelAttribute
- **异常处理**：@ControllerAdvice、@ExceptionHandler

### 4. Spring Boot
- **自动配置原理**：@EnableAutoConfiguration、@ConditionalOnClass、@ConditionalOnProperty
- **外部化配置**：application.properties、application.yml、环境变量、命令行参数
- **Profile机制**：不同环境的配置分离
- **监控管理**：Actuator端点、健康检查、指标收集

### 5. Spring Security
- **认证机制**：用户名密码、OAuth2、JWT
- **授权机制**：基于角色、基于权限、基于表达式
- **会话管理**：Session、Token、无状态
- **安全配置**：WebSecurityConfigurerAdapter、SecurityFilterChain

## 性能优化

### 1. IoC容器优化
- **Bean作用域选择**：合理使用singleton和prototype
- **延迟加载**：使用@Lazy注解
- **循环依赖避免**：重新设计依赖关系

### 2. AOP性能优化
- **切点表达式优化**：精确匹配，避免过于宽泛
- **代理选择**：优先使用JDK动态代理
- **通知方法优化**：避免在通知中执行耗时操作

### 3. Spring Boot优化
- **自动配置排除**：排除不需要的自动配置
- **组件扫描优化**：精确指定扫描包
- **外部化配置优化**：合理使用配置优先级

### 4. 数据库访问优化
- **连接池配置**：合理设置连接池参数
- **事务管理**：合理使用事务传播行为
- **缓存使用**：使用Spring Cache提高性能
