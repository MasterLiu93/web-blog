# Java集合框架总结

Java集合框架是Java语言中最重要的基础库之一，提供了丰富的数据结构实现。掌握集合框架对于后端开发至关重要，也是面试中的高频考点。

## 集合框架体系结构

Java集合框架主要分为三大类：**Collection**、**Map**和**工具类**。

### Collection接口体系

```
Collection
├── List (有序，可重复)
│   ├── ArrayList (数组实现，随机访问快)
│   ├── LinkedList (链表实现，插入删除快)
│   ├── Vector (线程安全，已过时)
│   └── Stack (栈，继承自Vector)
├── Set (无序，不可重复)
│   ├── HashSet (哈希表实现)
│   ├── LinkedHashSet (保持插入顺序)
│   └── TreeSet (红黑树实现，有序)
└── Queue (队列)
    ├── PriorityQueue (优先队列)
    ├── ArrayDeque (双端队列)
    └── LinkedList (也可作为队列)
```

### Map接口体系

```
Map
├── HashMap (哈希表实现，无序)
├── LinkedHashMap (保持插入顺序)
├── TreeMap (红黑树实现，有序)
├── Hashtable (线程安全，已过时)
├── ConcurrentHashMap (线程安全的HashMap)
└── WeakHashMap (弱引用HashMap)
```

## 核心面试重点

### 1. 集合选择原则
- **ArrayList vs LinkedList**：随机访问 vs 插入删除
- **HashMap vs TreeMap**：无序快速查找 vs 有序遍历
- **HashSet vs TreeSet**：去重 vs 有序去重

### 2. 线程安全问题
- **非线程安全**：ArrayList、HashMap、HashSet
- **线程安全**：Vector、Hashtable、ConcurrentHashMap
- **Collections.synchronizedXXX()**：包装成线程安全

### 3. 性能特点
- **时间复杂度**：增删改查的时间复杂度
- **空间复杂度**：内存占用情况
- **扩容机制**：动态扩容的实现原理

### 4. 常见陷阱
- **并发修改异常**：ConcurrentModificationException
- **内存泄漏**：集合中的对象引用
- **性能问题**：不当的集合选择

## 详细知识点

### List接口
- **ArrayList**：基于动态数组，适合随机访问
- **LinkedList**：基于双向链表，适合插入删除
- **Vector**：线程安全的ArrayList，已不推荐使用

### Set接口
- **HashSet**：基于HashMap实现，无序
- **LinkedHashSet**：保持插入顺序的HashSet
- **TreeSet**：基于TreeMap实现，有序

### Map接口
- **HashMap**：最常用的Map实现
- **LinkedHashMap**：保持插入顺序的HashMap
- **TreeMap**：基于红黑树的有序Map
- **ConcurrentHashMap**：线程安全的HashMap

### Queue接口
- **PriorityQueue**：基于堆的优先队列
- **ArrayDeque**：基于数组的双端队列
- **LinkedList**：也可作为队列使用

## 工具类

### Collections工具类
- **排序**：sort()、reverse()
- **查找**：binarySearch()
- **同步**：synchronizedXXX()
- **不可变**：unmodifiableXXX()

### Arrays工具类
- **排序**：sort()
- **查找**：binarySearch()
- **填充**：fill()
- **比较**：equals()、deepEquals()
